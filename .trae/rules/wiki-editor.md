# Wiki 编辑器规范

## 使用场景

| 场景       | 说明                     |
| ---------- | ------------------------ |
| 添加新功能 | 新功能开发完成后创建文档 |
| 修改旧功能 | 功能变更时更新文档       |
| 删除旧功能 | 功能废弃时标记文档状态   |
| 新建文档   | 创建 Wiki 文档时         |
| 更新文档   | 修改现有文档时           |
| 一致性检查 | 验证文档与代码一致时     |

## 操作流程

### 完整流程

```
明确任务 → 调用 project-wiki → 执行操作 → 一致性检查 → 更新 CHANGELOG → 设置状态
```

### 场景优化

| 场景       | 优化流程                                                                    |
| ---------- | --------------------------------------------------------------------------- |
| 添加新功能 | 明确功能 → project-wiki 生成 → 一致性检查 → 更新 CHANGELOG → 设置 Published |
| 修改旧功能 | 明确变更 → project-wiki 更新 → 一致性检查 → 更新 CHANGELOG → 保持状态       |
| 删除旧功能 | 明确废弃 → project-wiki 标记 → 更新 CHANGELOG → 设置 Deprecated             |

## 原子化操作

### 1. 生成文档

```
project-wiki → 选择模板 → 生成文档
```

说明：根据文档类型选择对应模板，生成后设置初始状态为 Draft

### 2. 更新文档

```
project-wiki → 修改内容 → 一致性检查
```

说明：修改内容后需验证与代码的一致性

### 3. 删除文档（标记废弃）

```
project-wiki → 标记 Deprecated → 更新 CHANGELOG
```

说明：不直接删除，而是标记状态并记录变更

### 4. 一致性检查

```
project-wiki → 版本核对 → 术语核对 → 数据核对
```

- 版本核对：文档版本号与代码版本一致
- 术语核对：术语表中的术语使用统一
- 数据核对：配置参数、API 端点与代码一致

### 5. 更新 CHANGELOG

```
添加版本 → 添加日期 → 添加变更类型 → 添加描述
```

变更类型：新增 / 修改 / 删除 / 修复

### 6. 状态管理

| 当前状态    | 操作     | 新状态      |
| ----------- | -------- | ----------- |
| -           | 新建     | Draft       |
| Draft       | 审核通过 | Published   |
| Published   | 持续更新 | Maintenance |
| Published   | 功能废弃 | Deprecated  |
| Maintenance | 归档     | Archived    |

## 注意事项

- 始终使用 project-wiki 作为文档工具
- 文档必须遵循 project-wiki 模板
- 发布前必须执行一致性检查
- 所有变更必须记录到 CHANGELOG.md
- 删除功能时仅标记状态，不删除文档
