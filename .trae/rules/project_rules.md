# 操作手册

## 主要原则

1. 主要使用中文
2. 代码和文档不同时修改
3. 文档编写自动调用 project-wiki
4. 代码编写自动调用 go-skill

## 文档编写

1. 使用 project-wiki 以 go-skill 辅助文档编写
2. 文档注意长度，对于复杂文档进行拆分。
3. 文档要注意目录结构、文章结构
4. 文档保持一致性：功能文档、需求稳定、架构文档、接口文档、数据文档等。
5. 所有文档修改，默认不需要兼容旧版本，如需兼容会特殊说明
6. 友好术语是面向用户使用，专业术语是面对插件开发和系统开发使用。

### 功能修改

1. 先理解修改内容
2. 调用技能生成修改方案
3. 查找当前项目相关文档
4. 对于wiki中所有文档进行覆盖式修改

## 代码编写

1. 使用 go-skills 编写代码
2. 遵循代码设计
3. 以文档驱动开发
4. 代码完成后要更新开发文档的进度
5. 编译代码为当前平台的可执行文件 `bin/chopsticks.exe`

### 添加修改功能

1. 理解处理的功能
2. 根据文档进行完成
3. 对于相关代码做出同步更改
4. 检查新编写的代码问题并修复

## 提交流程

### 提交修改内容

1. 根据修改内容进行总结，分成不同步骤
2. 根据内容分步骤暂存修改内容，逐步进行提交
3. 默认不进行推送，除非强调

### 更新版本

1. 参考ROADMAP.md，根据版本规划进行更新。
2. 更新CHANGELOG.md。
3. 添加tag。
4. 默认不进行推送，除非强调
