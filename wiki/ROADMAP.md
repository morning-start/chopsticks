# Chopsticks 路线图 (Roadmap)

> 使用敏捷开发方式规划项目演进路径

---

## 项目愿景

**Chopsticks（筷子）** 致力于成为 Windows 平台最灵活、开发者友好的包管理器，通过 JavaScript/Lua 脚本赋予开发者完全控制软件安装流程的能力。

---

## 敏捷开发规划

### 迭代周期

- **Sprint 周期**: 2 周
- **发布节奏**: 每个 Sprint 结束时发布一个版本
- **版本规范**: [Semantic Versioning](https://semver.org/lang/zh-CN/)

## 贡献指南

### 如何参与

1. **查看任务**: 查看 [GitHub Issues](https://github.com/your-repo/chopsticks/issues)
2. **选择任务**: 选择标有 `good first issue` 或 `help wanted` 的任务
3. **创建分支**: 从 `main` 分支创建功能分支
4. **提交 PR**: 完成任务后提交 Pull Request

### 开发规范

- 遵循 [Go 代码规范](../wiki/developer/STYLE.md)
- 编写单元测试
- 更新相关文档
- 遵循 [Conventional Commits](https://www.conventionalcommits.org/)
