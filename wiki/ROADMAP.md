# Chopsticks 路线图 (Roadmap)

> 成为 Windows 平台最好的包管理器

---

## 项目愿景

**Chopsticks（筷子）** 致力于成为 Windows 平台最好的包管理器，我们的目标是：

- 🏆 **成为 Windows 平台最好的包管理器**
- ⚡ **提供极致的安装体验**
- 🌱 **构建活跃的开发者社区**

---

## 敏捷开发规划

### 迭代周期

| 项目 | 说明 |
|------|------|
| **Sprint 周期** | 2 周 |
| **发布节奏** | 每个 Sprint 结束时发布一个版本 |
| **版本规范** | [Semantic Versioning](https://semver.org/lang/zh-CN/) |

### 优先级定义

| 优先级 | 说明 | 目标版本 |
|--------|------|----------|
| **P0** | 核心 MVP，必须完成 | v0.x |
| **P1** | 核心增强，重要功能 | v0.x |
| **P2** | 体验优化，提升易用性 | v0.x |
| **P2+** | 性能优化，极致性能 | v0.x |
| **P3** | 未来扩展，长期规划 | v1.x+ |

## 参与贡献

### 如何参与

我们欢迎各种形式的贡献，包括但不限于：

1. **代码贡献**
   - 查看 [GitHub Issues](https://github.com/chopsticks-bucket/main/issues)
   - 选择标有 `good first issue` 或 `help wanted` 的任务
   - 从 `main` 分支创建功能分支
   - 完成任务后提交 Pull Request

2. **文档贡献**
   - 完善用户文档
   - 编写教程和示例
   - 翻译文档

3. **测试贡献**
   - 报告 bug
   - 测试新功能
   - 编写测试用例

4. **社区贡献**
   - 回答问题
   - 分享使用经验
   - 推广项目

### 开发规范

- 遵循 [Go 代码规范](./developer/STYLE.md)
- 编写单元测试
- 更新相关文档
- 遵循 [Conventional Commits](https://www.conventionalcommits.org/)

### 贡献者权益

- 成为项目贡献者
- 获得贡献者徽章
- 参与项目决策
- 优先获取支持
